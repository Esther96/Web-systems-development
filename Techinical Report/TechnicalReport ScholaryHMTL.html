    • Abstract

    MusSmart is a music smart application developed in PHP and REST services that connect to Last.fm API in order to get the songs from the library and offers it to the user. A user can search for songs by artist, title, tags, genre and by lirics with the aim of ElasticSearch API. The user is very entertained by the option given, the application let him to save playlists, make his own playlist, add favorites, listen offline and many others.

    Introduction

    Music was always a way to entertain or to escape reality in a good way. Music is a part of our lives and with every occasion we are accompanied by songs. The humans behave in such a way they colors the life they live. Movies are covered in music, theaters, lessons, public transport and publicity and many others are related to music in order to have a great impact.

    Music smart assistant story begins with the need of customization of all music we listen or we want, even related with other apps from the phone or web. In order to be up-to-date with our favorite music, a smart assistant can make better recommendations, faster than us. So MusSmart stays for the music and smart, these two words makes everything perfect. This app is multi-platforms and gives the offline mode. Offline mode is required for the places with no internet and signal, and the application stores in xml libraries songs and the user can use it.

    The number of daily users we expected to grow every time because now we release the free version. The growth is shown even in the number of hardware platforms our clients run on, the number of development teams that work with our products, the number of external apps we host on our platform, the number of songs we have in our catalogue.

    We will make a lot of efforts in order to render at a minimum time the content wanted especially for the search and discover area, because those two are in changing and is vital for the user. As we say before, we offer the offline support. The offline area is available from the application to the device, mobile or desktop. Offline mode is required for the places with no internet or secure Wi-Fi or for no signal, like airplanes. So, the songs added to favorites, the history, and all the playlists saved/ album saved are available to listen offline by storing them into device, and synchronizing, and transfer from the app with a XML shareable playlist (XSPF).

    The search for offline will works for the stored songs, but some areas will be available when a connection to internet is made. The important past is that the user doesn’t have to download the songs in the device, because they are contained in application.

    Another aspect of this music smart assistant is Facebook integration. We want the user to be connected with the artists he likes and follows on Facebook. Also for upcoming events like festivals, concerts, immediately a song appear related to that event, it will be displayed to user in discover section of the application.

    Lyrics search and lyrics displayed on play is possible with MusSmart. We thought it would be really convenient to allow tracks to be searched by lyric query. When we talk about search, we’re talking about both the search query itself and the index of search terms. First, we build an index based on the open source search engine Elasticsearch (indeed the entire MusSmart search engine relies on it). We index JSON documents representing track lyrics. Querying lyrics text however is more akin to a traditional search on a text document where the frequency of the query must be taken into account. In order to compute a score that reflects the matching between the user query and the track lyrics, we use the ElasticSearch Function Score Query.

    Overview

    MusSmart is a music smart assistant that lets user to organize songs by preference, to make playlists, to add to favorites and play history and offers recommendations based on the user feedback. It has search options and possibility to listen offline. This application even plays past events/concerts attendance, near future events and provide a map of events.

    Fetching Last.fm Playlists

    Use the playlist.fetch method call to fetch XSPF playlists. XSPF is a web standard for sharable playlists, and XSPF libraries are available in several languages. playlist.fetch takes a 'lastfm' protocol URL as an argument; this identifies the playlist you're requesting. The following protocol url schemes are currently supported:

  • Album Playlists: lastfm://playlist/album/<album_id>
  • User Playlists: lastfm://playlist/<playlist_id>
  • Tag Playlists: lastfm://playlist/tag/<tag_name>/freetracks

    • Playback

    If you'd like to add play links to your playlist you can link to the Last.fm track page (which will be present in the XSPF extension node), appending '?autostart' to the URL.

    XSPF

    Provider: Xiph.Org Foundation

    API Documentation URL: http://www.xspf.org/quickstart/

    A listener community grown out of Audioscrobbler. Listeners send information about each track they play to Last.fm. That way Last.fm can build statistics about your music habits and make suggestions for music you might want to check out in the future.

    XSPF is the XML format for sharing playlists.

  • It is portable. You should be able to carry a playlist from one place to another.
  • It is well-engineered. Most playlist formats have bugs that make life harder for programmers and users.
  • It is free (as in liberty) and open. No proprietary lock-in.
