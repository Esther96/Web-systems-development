    • Abstract

    MusSmart is a music smart application developed in PHP and REST services that connect to Last.fm API in order to get the songs from the library and offers it to the user. A user can search for songs by artist, title, tags, genre and by lirics with the aim of ElasticSearch API. The user is very entertained by the option given, the application let him to save playlists, make his own playlist, add favorites, listen offline and many others.

    MusSmart application want to organize the music someone listen to in a smart way, to give recommendation, to make charts and show events in order to create a better experience for the user.

  • api_key: Web app 32-character API Key.
  • token: The authentication token received at your callback url as a GET variable.

    Last.fm API

    Last.fm API is a powerful API that let developers to connect with it and Last.fm pull music data and user data from this social music database. The application we talk about in this report is MusSmart. MusSmart use the web services of Last.fm and create an environment for the end user to access music, to create playlists, in a word, to “manage” the music he want. Last.fm API use cases include music data visualization and analysis. So it can make recommendations, has feature for events and social groups (friends). The data it is not copied, it just used with secure credentials and the user accounts are authorized by this services. For the PHP application Last.fm provide PHP Last.fm API.

    Provider: Last.fm

    Link: https://www.last.fm/

    API Documentation URL: https://www.last.fm/api

    Last.fm puts at disposition to the developer/user of API, a set of methods that includes:

  • Album: search, information, and possibility for add, remove and get tops.
  • Artist: search, add, remove, similar artists, top albums, tracks.
  • Authentication (auth.getMobileSession, auth.getSession, auth.getToken)
  • Chart: for top artists, top tracks and top in general
  • Geo for top artists and tracks
  • Library: as a collection of artists
  • Tag for getting information, get top albums, top artists, top tracks and weekly chart list and similar songs.
  • Track: add to favorites (track.love method), remove (track.unlove method), search, scrobble and update now playing.
  • User (all the preferences and information: friends, artists, playlists, album, tracks: favorites, recent, personal and many other methods)

    • The most recognizable aspect of the Last.fm is the scrobbling, meaning to use Last.fm to keep track of all the songs one has played on their music service. This features are used by other popular app in connection with Last.fm. Last.fm's scrobbler feature can be used from the official desktop or mobile apps. Music you listen to will then be tracked and automatically added to your music profile. Last.fm can be integrated with other popular music services, including your iTunes library, Spotify, YouTube, SoundCloud and many others. This means that you can also scrobble music you listen to from third-party apps like these when you have Last.fm integrated.

    The "Now Playing" service lets a client notify Last.fm that a user has started listening to a track. This does not affect a user's charts, but will feature the current track on their profile page, along with an indication of what music player they're using. This API method call is optional for scrobbling clients, but recommended. Requests should be sent as soon as a user starts listening to a track.

    Sending a Request

    The web service method for sending Now Playing information is track.updateNowPlaying. Once a request has been received by Last.fm the following sequence of events takes place on our side:

  • Check that the request passes our filters.
  • Find the track in our catalogue.
  • Update the user's Now Playing status with the track and client.
  • Return a response to the client indicating the outcome of the request.

    • The request must be authenticated and the flow looks like that.

    Another aspect that deserve our attention is the playlists API. Last.fm playlists do not contain streaming content. So, there is need for fetching, used with the playlist.fetch method call in order to fetch XSPF playlists. XSPF is a web standard for sharable playlists, and XSPF libraries are available in several languages. playlist.fetch takes a 'lastfm' protocol URL as an argument; this identifies the playlist you're requesting. The following protocol url schemes are currently supported:

  • Album Playlists: lastfm://playlist/album/<album_id>
  • User Playlists: lastfm://playlist/<playlist_id>
  • Tag Playlists: lastfm://playlist/tag/<tag_name>/freetracks

    • RADIO

    The radio API requires authentication. Once authenticated, you can tune the radio using the radio.tune API method. This takes a station parameter that must correspond to a last.fm protocol station url. Here is the general scheme of these URLs: lastfm:////

    Here is a list of the station types currently publicly available, with example protocol URLs:

      Artist
    • Similar Artists Radio (e.g. lastfm://artist/cher/similarartists )
    • Top Fans Radio (e.g. lastfm://artist/cher/fans )
      User
    • Library Radio (e.g. lastfm://user/last.hq/library )
    • Mix Radio (e.g. lastfm://user/last.hq/mix )
    • Recommendation Radio (e.g. lastfm://user/last.hq/recommended )
    • Neighbours Radio (e.g. lastfm://user/last.hq/neighbours )
      Tag
    • Global Tag Radio (e.g. lastfm://globaltags/disco )
    Fetching Radio Content Once the station is tuned, our application use the API method radio.getPlaylist to fetch content in XSPF format. Last.fm recommands to periodically pull on this XSPF service as it will provide content in small chunks. We recommend you prefetch a new XSPF before reaching the end of the last.

    XSPF is the XML format for sharing playlists.

  • It is portable. You should be able to carry a playlist from one place to another.
  • It is well-engineered. Most playlist formats have bugs that make life harder for programmers and users.
